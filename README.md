An interactive Streamlit application demonstrating **Reasoning-based Retrieval Augmented Generation (RAG)** without Vector Databases or Document Chunking. This project leverages the `pageindex` SDK and Groq's blazing fast `llama-3.3-70b-versatile` LLM.

## 🔑 The Core Concept

> **Traditional RAG** → chunk → embed → cosine similarity → retrieve
> **TreeRAG (PageIndex)** → build document tree → LLM reasons over the Table of Contents → retrieve exact, intact sections

**Why is this better?**
In Traditional RAG, Similarity ≠ Relevance. A chunk might just share keywords. In TreeRAG, an LLM acts like an expert scanning a book's index, pulling out whole contextual chapters and eliminating hallucinations caused by fragmented data.

## ⚠️ Constraints & Ideal Use Cases

TreeRAG relies on the premise that it can read and map the *natural hierarchy* of a document. Therefore, the accuracy heavily depends on the structure of your PDF.

**✅ Highly Accurate PDFs (Recommended):**
- **Research Papers:** Strict hierarchies (Abstract, Introduction, Methodology).
- **Financial Reports / 10-Ks:** Highly sectioned and itemized (MD&A, Risk Factors).
- **Technical Manuals & Textbooks:** Clear chapters and table of contents.
- **Legal Contracts:** Numbered sections and clauses.

**❌ Less Accurate PDFs (Constraints):**
- **Unstructured / Free-flowing text:** Novels or light essays without section headers.
- **Purely visual PDFs:** Scanned images without OCR or heavy diagram-based presentations.
- **Scattered layouts:** Marketing brochures or magazines with non-linear column reading paths.

## ✨ Features
1. **Direct PDF Upload:** Simply drag and drop any PDF. The app dynamically uploads it, processes its structure into a hierarchical tree, and prepares it for answering.
2. **Intelligent Router:** Ask specific questions about the document and it retrieves exact cited sections. Say "Hi" or Ask a general question, and it responds with normal conversational chat!
3. **Transparent Reasoning:** Watch the LLM's "thinking logic" as it navigates the document tree to pinpoint the exact context.
4. **Internal Tree Visualization:** Scroll through the raw JSON mapping generated by PageIndex directly in the UI to see exactly how the AI understands your document layout.

## 🚀 Getting Started

### 1. Install Dependencies
Make sure you have Python installed, then run:
```bash
pip install -r requirements.txt
```

### 2. Set Up Environment Variables
Create a `.env` file in the root directory (do not commit this file) and add your API keys:
```env
PAGEINDEX_API_KEY=your_pageindex_api_key_here
GROQ_API_KEY=your_groq_api_key_here
```
- Get PageIndex Key: [dash.pageindex.ai](https://dash.pageindex.ai/)
- Get Groq Key: [console.groq.com](https://console.groq.com/)

### 3. Run the App
Launch the Streamlit interface:
```bash
streamlit run app.py
```

### 4. How to Use
1. **Upload your PDF** in the main section of the app.
2. Wait a few moments as the AI scans and structures the document.
3. **View the Tree:** Open the "🌳 View Internal Tree Structure" expander to inspect the extracted document hierarchy in a neat, scrollable box.
4. **Start chatting!** You can say "hello" to test the conversational logic, or immediately drill into a highly technical query about the PDF you uploaded. 
5. Read the precisely cited answer along with the AI's transparent reasoning trace.
